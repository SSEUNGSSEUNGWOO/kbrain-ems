당신은 공공 교육 사업의 **차수별 결과보고서 초안**을 작성합니다.
운영자(승우님 외 6~7명)가 검수만 하면 되는 수준의 정돈된 마크다운을 출력하는 게 목표입니다.

## 입력

사용자 인자: `$ARGUMENTS`

- 빈 문자열이면 "cohortId 또는 이름을 알려주세요" 라고 묻고 멈춥니다.
- UUID 형식(36자, 하이픈 포함)이면 그대로 사용.
- 그 외에는 이름으로 간주, 짧은 일회용 스크립트로 `cohorts` 테이블을 `ilike '%이름%'` 매칭해 UUID 획득. 후보가 여러 개면 목록을 보여주고 멈춥니다.

## 워크플로우

### 1) 데이터 수집

```bash
cd frontend && bun run scripts/cohort-report-data.ts <UUID> --out ../reports/<slug>-data.json
```

`<slug>`는 `cohort.name`을 안전한 파일명으로 변환한 값.
규칙: 공백 → `-`, `/`·`\`·`:`·`*`·`?`·`"`·`<`·`>`·`|` 제거.
예: `전문인재 26-1기` → `cohort-전문인재-26-1기`

스크립트는 stderr로만 `saved → ...` 메시지를 출력합니다. 정상 종료 확인 후 다음 단계로.

### 2) 마크다운 작성

`reports/<slug>-data.json`을 읽고, 아래 표준 섹션 구조로 `reports/<slug>.md`를 작성합니다.
**모든 수치는 JSON에서 그대로 가져오기 — 추측 금지.** 데이터 없는 섹션은 `> 본 차수 미운영` 한 줄로 표시.

#### 표준 섹션

```
# {cohort.name} 결과보고서

## 1. 기수 개요
- **기수명**: {name}
- **카테고리 / 운영방식**: {category} / {delivery_method}
- **운영기간**: {started_at} ~ {ended_at}
- **모집기간**: {application_start_at} ~ {application_end_at} (없으면 생략)
- **정원 / 등록**: {max_capacity}명 / {enrollment.enrolled}명
- **지원·선발·등록**: 지원 {applicants} → 선발 {selected} → 등록 {enrolled}
- (트랙이 있으면) **트랙별 인원**: {tracks 표}

## 2. 운영 현황 — 출결
- **총 회차**: {sessions.length}
- **전체 출석률**: {attendance_overall.present_rate}% ({present_count} / {total_records})
- **회차별 출석표** (회차·일자·주제·강사·출석률):

| 회차 | 일자 | 주제 | 강사 | 출석률 |
|---|---|---|---|---|
| 1 | 2026-05-07 | [기술교육] 1회차 | 안진희, 신성진 | 100% |
| ... |

회차 attendance.rate가 null(출결 미입력)이면 `미입력`으로 표기.

## 3. 강사 운영
| 강사 | 등급 | 담당 회차 | 시간 | 만족도 평균 (n) |
|---|---|---|---|---|
| 안진희 | 1급 | 3 | 9.0 | 4.52 (24) |
| ... |

만족도 데이터 없으면 `-`.

## 4. 학습자 만족도
설문 한 개씩 다음 형식으로:

### 4-{i}. {survey.title}
- **응답률**: {response_rate}% ({n} / {denominator})
- **섹션별 평균** (척도 1~5):
  | 섹션 | 평균 | n |
  - 섹션 평균은 questions를 section_no로 묶어 likert5만 평균 계산.
- **강사별 만족도**: 위 강사 표에 이미 반영. 본 섹션엔 표기 생략.
- **주관식 응답** (text 문항 중 text_responses 있는 것):
  - 5건 이하면 모두 인용 (한 줄 인용 `> ...`).
  - 5건 초과면 대표 5건 인용 + 한 줄 요약 추가.

척도 평균은 소수점 2자리.

## 5. 진단 결과
- **사전**: 평균 {pre.avg}점 (n={pre.n}, σ={pre.sd})
- **사후**: 평균 {post.avg}점 (n={post.n}, σ={post.sd})
- **변화량**: {delta}점

pre/post 어느 한쪽이라도 null이면:
> 본 차수에는 사전·사후 진단 응답이 충분히 수집되지 않았습니다.

## 6. 과제 운영
| 과제 | 마감일 | 제출 / 대상 | 제출률 |

assignments 없으면 `> 본 차수 미운영`.

## 7. 종합 총평 및 시사점

3~5문장으로 다음 관점을 종합:
- 운영 성과 (출석률·등록·이수 흐름)
- 강사 운영 효과성 (만족도·구성)
- 학습자 변화 (진단 사전·사후 변화량이 있다면)
- 향후 개선 포인트 1~2개 (낮은 점수 항목·주관식 부정 의견에서 도출)

**톤**: 단정적 표현 지양. "...로 나타났다", "...로 보인다", "향후 ...할 필요가 있다".
**금지**: 데이터에 없는 추측, 사람 이름 거론, 평가.

## 8. 사진
> 회차별 활동 사진은 운영자가 별도 삽입.

---
*초안 자동생성: {generated_at}*
```

### 3) 결과 안내

작성 완료 후 다음을 사용자에게 보여줍니다:

```
✓ 보고서 초안 생성됨
  데이터: reports/<slug>-data.json
  마크다운: reports/<slug>.md

검수 후 PDF로 변환:
  bunx md-to-pdf reports/<slug>.md
```

## 주의

- 표 안에 강사 이름이 여러 명이면 `, `로 연결.
- 출석률·만족도 평균 자릿수는 JSON에 이미 반올림돼 있음 — 추가 가공 금지.
- 주관식 응답에 개인정보(이름·소속)가 보이면 그대로 인용하지 말고 `[학습자]`로 마스킹.
- 마크다운 표는 GitHub Flavored Markdown 규격.
