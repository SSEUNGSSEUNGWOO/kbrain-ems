03

EMS 기능 상세

Q1 즉시 착수 5월 운영 시작 전후 필수 12개
# 기능 설명
1 강사료 자동 산정·검증 강사 등급표(특급~3급) 매핑, 시간당 단가·일일 한도 자동 검증, 초과 시 차단
2 차수별 결과보고서 자동 초안 출결·만족도·사진 메타데이터 → 표준 템플릿 자동 채움 → 요약·총평 생성
3 표준 신청 웹폼 + 유효성 검증 확인 필요 필수항목·형식·중복신청 자동 차단, 접수증 자동 발송
* 신청자가 직접 사용하는 기능 — EMS 내 공개 페이지 vs 별도 앱 분리 결정 필요
4 사전 자가진단 자동 채점·트랙 추천 확인 필요 신청 시 자가진단 → 점수 계산 → 적합 트랙 1/2/3 자동 추천
* 신청자가 직접 사용하는 기능 — EMS 내 공개 페이지 vs 별도 앱 분리 결정 필요
5 신청자 자동 점수화 선발 평가표(직급·관련업무·자가진단·동기) 룰 기반 정량화
6 기관별·유형별 쿼터 자동 검증 중앙/지자체/공공 비율, 동일기관 한도, 트랙별 정원 자동 체크
7 입과안내 자동 발송 D-7 입과안내 / D-3 사전이수 미완료 알림 / D-1 리마인드 / D-day 접속안내
8 출결 자동 기록 비대면(Zoom 참여 로그) + 대면(QR 체크인) 통합, 실시간 모니터링
9 수료 판정·증서 발급 출석률+평가 룰 → 자동 판정 → PDF 수료증 일괄 생성·발송
10 설문 자동 집계·시각화 확인 필요 차수 종료 즉시 응답률·평균·문항별 분포 자동 산출, 차트 4종 생성
* 교육생이 설문 응답하는 페이지 필요 — EMS 내 공개 페이지 vs 별도 앱 분리 결정 필요
11 사전·사후 자동 비교·향상도 확인 필요 동일 문항 매칭 → 개인별·트랙별 향상도 자동 계산, KPI 보고 직접 활용
* 교육생이 진단에 응답하는 페이지 필요 — EMS 내 공개 페이지 vs 별도 앱 분리 결정 필요
12 회차별 누적 대시보드 만족도·향상도·신청선발·수료인증 4개 패널, 매 차수 자동 누적
Q2 예약 추진 6~7월 운영 안정화 시점 7개
# 기능 설명
1 평가위원 분배·집계 위원별 익명 코드 → 평가지 자동 분배 → 평균·표준편차·이상치 자동 집계
2 강사 일정 자동 알림 D-7 자료요청 / D-3 확인 / D-1 리마인드 / D-day 안내 자동 발송
3 실습환경 사전 점검 망분리 환경 접속·계정·데이터 상태 자동 점검, 차수 전일 리포트
4 부정 피드백 즉시 알림 평균 4.0 미만, 부정응답 30% 이상, 동일강사 연속 저평가 시 즉시 알림
5 트랙·직급별 자동 세분 향상도를 트랙·직급·기관유형별 피벗, 차수 누적 비교 차트 자동 갱신
6 미응시자 자동 리마인드 사전·사후·활용현황조사 미응답자 식별 → 단계별 리마인드
7 사업 진척률·KPI 대시보드 WBS 기반 진척률 + 산출물 상태 + 부문별 KPI + 리스크 요약
Q3 경량 처리 노코드·간단 워크플로우 3개
# 기능 설명
1 신청 상태 자동 알림 접수 → 검토 → 승인/거절 단계별 SMS·메일 자동 발송
2 선발 결과 일괄 통보 합격/예비/탈락별 템플릿 매칭 + 일괄 발송
3 리스크·이슈 보드 7대 리스크 등록부 + 이슈 라이프사이클 시각화 + 미해결 에스컬레이션

별도 도구 (5개)

EMS와 데이터를 주고받되, 파일 입출력·LLM 처리 중심으로 시스템 성격이 달라 독립 구축하는 항목.

Q1 즉시 운영 시작과 동시에 필요 2개
01 산출물 최종 오류 검수 LLM
NIA 송부 전 6종 오류 자동 검출 — 표기 통일성, 숫자 일치, 날짜 정합성, 인명·기관 오타, 산식 오류, 누락 항목
구현: Python + Claude API + python-docx/openpyxl
02 텍스트 응답 LLM 요약·분류 확인 필요
만족도 주관식 응답 → 감성분류(긍정/중립/부정) + 주제군집(강사/교재/환경/시간) + 핵심문장 추출. EMS 만족도 모듈에 포함할지, 별도 도구로 뺄지 확인 필요.
구현: Claude API
Q2 예약 6~7월 안정화 시점 2개
03 공문·안내문 템플릿 엔진
변수 치환 + LLM 톤 보정, 모집공고·입과안내·수료통보 등 8종 표준 템플릿 자동 생성·검수
구현: Python + HWP/Word 템플릿
04 회의록 정리 (STT + 요약)
Zoom/Teams 녹취 → STT → 요약·액션아이템·결정사항 자동 분류
구현: Whisper API + Claude API
Q3 경량 간단 워크플로우 1개
05 주간보고서 자동 합본
부문별(교육·챔피언·컨설팅·진단·시스템) 표준 입력폼 → 매주 금요일 자동 합본 PDF
구현: Python + PDF 생성

확인 필요 사항

의사결정 필요 항목

아래 항목은 구현 착수 전 확인이 필요합니다.

  • 텍스트 응답 LLM 요약·분류 — EMS 만족도 모듈 내 포함 vs 별도 도구 분리
  • 외부 사용자 페이지 위치 — 아래 4개는 교육생/신청자가 직접 접근하는 페이지. EMS 내 공개 페이지로 포함할지, 별도 앱으로 분리할지:
      · 신청 웹폼 (신청자)
      · 자가진단 (신청자)
      · 만족도 설문 (교육생)
      · 사전·사후 진단 (교육생)
      A) EMS 안에 외부 공개 페이지로 포함 (/apply, /survey, /diagnosis)
      B) 별도 공개 앱으로 분리하고 EMS는 데이터만 수신
  • 기술 스택 최종 확정 — Next.js 16 + Neon PostgreSQL + Drizzle ORM + shadcn/ui 예정
  • 인증 방식 — 운영자 전용 간단 인증(쿠키) vs Supabase Auth(Google OAuth)
  • 알림 발송 채널 — 메일 / SMS / 카카오 알림톡 중 어디까지 지원할지
  • 설문·진단 실시 주기 — 만족도 설문과 사전·사후 진단을 어떤 주기로 실시하는지 확인 필요:
      · 만족도: 매 차수 종료 시? 과목별? 중간+최종?
      · 사전진단: 입과 전 1회?
      · 사후진단: 수료 직후 1회?
      · 활용현황조사: 수료 3개월 후?
      → 주기에 따라 설문 페이지 설계·자동 발송 로직이 달라짐
